01 — fullstack.md

What is Full Stack
Development?

// Understanding the complete picture — from browser to database and back.

📖
Definition
A full stack developer is proficient in both frontend (client-side) and backend (server-side) development. The term "stack" refers to the collection of technologies used to build a complete web application — from what the user sees in their browser, to how data is processed and stored on servers.

Full stack development encompasses everything: UI/UX, business logic, databases, APIs, infrastructure, security, and performance optimisation.
🔄
The Request-Response Cycle
1. User types a URL / clicks a button
2. Browser sends an HTTP request to the server
3. Server processes the request (routing, auth, logic)
4. Database is queried for relevant data
5. Response (HTML/JSON) is returned to the browser
6. Browser renders the content for the user

HyperText Markup
Language

// The skeleton of every webpage — structure, semantics, and meaning.

📄
What is HTML?
HTML is a markup language that defines the structure and meaning of web content. It uses nested elements enclosed in angle-bracket tags to describe headings, paragraphs, links, images, forms, and more.
🏷️
Semantic HTML
Semantic elements like <header>, <nav>, <main>, <article>, and <footer> convey meaning to browsers, search engines, and assistive technologies — not just visual structure.
♿
Accessibility
Proper HTML structure — alt attributes on images, <label> on form fields, logical heading hierarchy — is foundational to building accessible web experiences for all users.

03 — styles.css

Cascading Style
Sheets

// The language of visual design — controlling colour, layout, typography, and motion.

🎯
The Cascade
CSS resolves style conflicts through specificity, inheritance, and source order. Inline styles beat IDs, which beat classes, which beat elements. !important overrides all.
📦
Box Model
Every element is a rectangular box with content → padding → border → margin. box-sizing: border-box makes padding and border part of the element's total size.
🔧
CSS Variables
Custom properties --primary: #ff6b6b defined on :root allow centralised theming — change one value and it propagates everywhere.

// The world's most popular CSS framework — responsive, component-rich, and production-ready.

🏗️
What is Bootstrap?
Bootstrap is a free, open-source CSS framework that provides a 12-column responsive grid system, pre-built UI components (buttons, navbars, modals, cards), and utility classes. It allows rapid, consistent, mobile-first UI development without writing CSS from scratch.

Loaded via CDN: <link rel="stylesheet" href="bootstrap.min.css">
📐
The Grid System
Bootstrap divides the page into 12 columns per row. Responsive breakpoints allow different layouts at different screen widths:

xs <576px · sm ≥576px
md ≥768px · lg ≥992px
xl ≥1200px · xxl ≥1400px

05 — app.js

JavaScript

// The behaviour layer — making pages interactive, dynamic, and alive.

🔤
Variables & Types
let and const (ES6+) replace var. JS is dynamically typed: strings, numbers, booleans, arrays, objects, null, undefined, and Symbol.
🌲
DOM Manipulation
The Document Object Model is a live tree of HTML elements. JS can querySelector, read/set properties, add/remove classes, and create/delete nodes dynamically.
🔗
Fetch API & Async
fetch() with async/await makes non-blocking HTTP requests. Promises chain with .then() / .catch() for clean asynchronous code.

JavaScript Evolution

ES5 — 2009
Foundation
JSON.parse, Array.map/filter/reduce, strict mode. The baseline all browsers support.
ES6/ES2015
Modern JavaScript
let/const, arrow functions, template literals, classes, destructuring, Promises, modules (import/export).
ES2017+
Async/Await
async/await syntax for cleaner asynchronous code, object spread (...), optional chaining (?.), nullish coalescing (??).
ES2023+
Current Standard
Array .toSorted(), .with(), hashbang syntax, WeakMaps — plus ongoing TC39 proposals for future features.
